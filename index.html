Role, 속성, 상태

Role속성 / 상태요소설명
button div, a
  • 요소를 버튼 위젯으로 식별
  • 접근 가능한 이름은 요소의 텍스트 콘텐츠로 정의
tabindex="0" div, a
  • 탭 순서에 요소를 포함
  • href 속성이 없는 a 요소에 필수
aria-pressed="false" a
  • 토글 버튼으로 식별
  • 버튼이 눌리지 않은 상태임을 전달
aria-pressed="true" a
  • 토글 버튼으로 식별
  • 버튼이 눌린 상태임을 전달

네이티브 vs ARIA

네이티브
<button disabled>확인</button>
warning 포커스 불가, 폼 전송 제외
ARIA
<button aria-disabled="true">확인</button>
check_circle 포커스 가능, 스크린리더에 비활성화 전달

키보드 인터랙션

KeyAction
Tab버튼으로 포커스 이동
Shift + Tab이전 요소로 포커스 이동
Enter / Space버튼 활성화

Do / Don't

check_circle Do
<button type="button"> 확인 </button>
네이티브 button 요소 사용 권장
cancel Don't
<div class="btn" onclick="submit()"> 확인 </div> <!-- role, tabindex, 키보드 이벤트 누락 -->
div 요소를 버튼으로 사용할 경우 role, tabindex, 키보드 이벤트를 반드시 추가해야 합니다.

접근성 체크리스트

<button> 네이티브 요소 사용 또는 role="button" 적용
Enter와 Space로 활성화 가능
포커스 표시가 시각적으로 명확 (SC 2.4.7)
비활성화 시 aria-disabled="true" 또는 disabled 속성 적용
접근 가능한 이름 제공 (텍스트 레이블 또는 aria-label)
터치 대상 크기 최소 24×24px (SC 2.5.8)

<!-- 기본 버튼 -->
<button type="button">확인</button>

<!-- 아이콘 포함 버튼 -->
<button type="button">
  <svg aria-hidden="true">...</svg>
  전송
</button>

<!-- 아이콘 전용 버튼 -->
<button type="button" aria-label="닫기">
  <svg aria-hidden="true">...</svg>
</button>

<!-- 비활성화 버튼 -->
<button type="button" disabled>비활성화</button>

<!-- div를 버튼으로 사용 (비권장) -->
<div role="button" tabindex="0">확인</div>
